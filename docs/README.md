# qresid manual

This directory contains tutorial material for the GitHub repository version of
`qresid`. The installed Stata package remains minimal: `qresid.ado` and
`qresid.sthlp`.

## Start here

- [Markdown reference](reference.md)
- [Features and diagnostic workflow](features.md)
- [Statistical background](statistical-background.md)
- [Validation and benchmarking](validation-benchmarking.md)
- [Project scope](project-scope.md)

Start with the reference page for syntax and stored results, then use the
background, validation, and scope pages when you want the mathematical and
release context behind the examples.

## Model choice tutorials

- [Continuous outcomes](continuous.md)
- [Binary and binomial outcomes](binary-binomial.md)
- [Count outcomes](counts.md)
- [Link and functional-form checks](link-functional-form.md)
- [Zero-inflated, truncated, and censored counts](special-counts.md)
- [Hurdle count models](hurdle.md)
- [Frequency weights and dispersion replay](weights-dispersion.md)

Each tutorial follows the same rhythm: an applied diagnostic question, the
fitted Stata models, selected output, figures, interpretation, and a short
take-home message. The goal is to teach how the same normal-scale residual
language can be used across model classes without ignoring each model's fitted
probability law.

## Reproducing figures and output

The tutorial figures and short Stata output excerpts are generated by:

```stata
do docs/scripts/build_manual_assets.do
```

Run the command from the `qresid` repository root. The script writes figures to
`docs/assets/img/` and output excerpts to `docs/assets/output/`.
